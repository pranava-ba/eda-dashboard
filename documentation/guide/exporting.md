# Exporting Reports

When you've found something worth sharing, you can save the current view as a document straight
from the top bar — no screenshots needed.

## The two buttons

In the top-right corner:

⬇ PDF
: Saves the current screen as a **PDF** — ideal for emailing, printing, or attaching to a report.
On the desktop app you choose where to save it; in the web demo it opens your browser's
**Print → Save as PDF** dialog.

⬇ HTML
: Saves a self-contained **HTML** file you can reopen in any browser. Handy for keeping an
interactive-looking snapshot on disk.

## What gets captured

The export reflects **what's on screen right now** — the active tab, its charts, tables, and
test results, using your current theme. So set things up the way you want them first:

1. Choose **Count** or **Proportion** and **With / Without target** in the top bar.
2. Open the tab and select the variables you want to show.
3. Then click **⬇ PDF** or **⬇ HTML**.

```{admonition} Tip — one analysis per export
:class: tip
Exports capture a single view, so produce one document per finding (e.g. "Income vs Churn"),
rather than trying to fit everything into one page. Toolbars and buttons are automatically hidden
from the output so it looks clean.
```

## Light or dark

The report uses whichever **theme** is active. The light theme generally prints and reads better
on paper; toggle it with the 🌙 / ☀️ button before exporting if needed.

That's the full workflow — from a raw file to a shareable report. If any term still isn't clear,
the {doc}`../glossary` collects them all in one place.
